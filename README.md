# EVE 预警系统

- **注意在EVE晨曦可能违反用户协议造成的任何损失由使用者自行承担**

基于 .NET 8.0 的 EVE Online 游戏智能预警系统，通过图像识别和实时网络通信，帮助玩家监控游戏状态并及时预警。

## 功能特性

### 客户端
- **屏幕实时监控**：3秒倒计时抓取游戏窗口截图
- **智能图像识别**：
  - 使用 Emgu.CV (OpenCV) 进行模板匹配
  - 5种预警色块检测（宣战、对战、糟糕、不良、白名）
  - 灰度匹配 + 颜色验证的双重识别策略
- **OCR 文字识别**：自动识别玩家角色名（PaddleOCRSharp）
- **实时网络通信**：
  - TCP Socket 连接服务器
  - 预警数据实时推送
  - 自动重连机制（最多5次，递增延迟）
  - 心跳保活（30秒间隔）
  - 预警去重，避免重复推送
- **灵活的 UI 操作**：
  - 拖动滑动条裁切检测区域
  - 框选裁切工具可视化选择区域
  - 实时日志记录
  - 预警声音提示
  - 可配置更新频率

### 服务器端
- **多客户端管理**：支持多个客户端同时连接
- **实时消息广播**：接收预警并广播给其他在线玩家
- **在线用户监控**：显示用户名、星系、连接时间、最后心跳
- **心跳检测**：自动清理超时客户端（30秒间隔，3次超时判定）
- **优雅退出**：服务器关闭时断开所有客户端

## 快速开始

### 环境要求

- Windows 10/11
- .NET 8.0 SDK
- Visual Studio 2022（推荐）

### 编译运行

1. **克隆项目**
   ```bash
   git clone <repository-url>
   cd EVE预警
   ```

2. **打开解决方案**
   ```
   使用 Visual Studio 打开 EVE预警.sln
   ```

3. **还原依赖**
   ```bash
   dotnet restore
   ```

4. **编译项目**
   ```bash
   dotnet build
   ```

5. **运行**
   - 先启动 **EVE预警服务器** 项目
   - 再启动 **EVE预警** 客户端项目

### 使用流程

1. **启动服务器**
   - 运行服务器程序
   - 点击"启动服务器"按钮，默认监听 2424 端口

2. **配置客户端**
   - 输入服务器 IP 地址
   - 输入当前星系名称

3. **抓取游戏窗口**
   - 切换到 EVE 游戏窗口
   - 点击"抓取窗口"按钮，3秒后自动截图

4. **选择检测区域**
   - 使用滑动条调整区域，或点击"框选裁切"可视化选择
   - 确保区域覆盖预警图标显示位置

5. **开始检测**
   - 点击"开始检测"按钮
   - 系统会定期检测并发送预警数据

## 网络协议

### 消息格式

所有消息使用 JSON 格式，以换行符 `\n` 分隔。

**注册消息**
```json
{
  "type": "register",
  "username": "玩家名称",
  "galaxy": "星系名称"
}
```

**心跳消息**
```json
{
  "type": "heartbeat"
}
```

**预警消息**
```json
{
  "type": "alert",
  "galaxy": "星系名称",
  "alerts": ["RedStar:2", "OrangeCross:1"]
}
```

**广播消息（服务器→客户端）**
```json
{
  "galaxy": "星系名称",
  "message": "宣战:2,对战:1"
}
```

## 开发说明

### 添加新的预警类型

1. 在 `ImageProcessor.cs` 的 `AlertType` 枚举中添加新类型
2. 添加对应的模板图片（如 `6.png`）
3. 在 `Main.cs` 的 `alertTypeMap` 中添加中文映射
4. 在 `GetAlertMessage` 方法中添加对应的提示信息

### 自定义模板匹配阈值

在 `ImageProcessor.cs` 第 216 行修改匹配阈值：
```csharp
if (maxVal >= 0.85)  // 修改此值调整匹配灵敏度
```

## 注意事项

- 确保 EVE 游戏窗口未被遮挡
- 检测区域应仅包含预警图标区域
- 服务器端口默认为 2424，可在服务器端修改
- 客户端自动重连最多尝试 5 次
